Extrait de code - Connexion a la base de donnees

PHPconnexion.php
// Connexion a la BDD avec PDO
// J'utilise PDO plutot que mysqli parce que c'est plus facile
// a utiliser avec les requetes preparees
try {
    $pdo = new PDO(
        'mysql:host=localhost;dbname=egnaro;charset=utf8mb4',
        $user,
        $password,
        // Mode exception pour avoir des messages d'erreur clairs
        [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION]
    );
} catch (PDOException $e) {
    // On log l'erreur dans un fichier plutot que de l'afficher
    // pour pas montrer d'infos sensibles aux utilisateurs
    error_log('Erreur BDD : ' . $e->getMessage());
    die('Erreur de connexion.');
}

Probleme rencontre

Apres avoir deploye le site sur le serveur, j'avais une erreur 403 Forbidden. J'ai mis du temps a comprendre que le probleme venait des permissions du fichier .htaccess et aussi de la directive AllowOverride qui etait desactivee dans la config Apache.

Ce que j'ai fait : j'ai corrige les permissions avec chmod 644 .htaccess et j'ai active AllowOverride All dans le VirtualHost. Apres ca, le site s'affichait normalement.

Mise en production

Voici les etapes qu'on a suivies pour mettre le site en ligne :

  1. Preparation du serveur - Installation d'Apache, PHP et MySQL sur la machine Debian. Configuration du VirtualHost.
  2. Transfert des fichiers - Au debut avec FileZilla, puis on a mis en place un depot Git pour faciliter les mises a jour.
  3. Base de donnees - Import du fichier SQL, creation d'un utilisateur dedie avec des droits limites.
  4. Tests - On a verifie que toutes les pages fonctionnent, que les formulaires envoient bien les donnees, etc.
  5. Documentation - J'ai fait un petit guide d'installation pour qu'un autre etudiant puisse refaire la meme chose.

Extrait de code - Menu burger pour le mobile

JavaScriptscript.js
// Gestion du menu burger en version mobile
// Quand on clique dessus, on affiche/cache le menu
const burger = document.querySelector('.burger');
const nav = document.querySelector('.nav-links');

burger.addEventListener('click', () => {
    nav.classList.toggle('active');
    burger.classList.toggle('toggle');
});

// Petit bug que j'ai eu : le menu restait ouvert quand
// on cliquait sur un lien. Du coup j'ai ajoute ca :
nav.querySelectorAll('a').forEach(link => {
    link.addEventListener('click', () => {
        nav.classList.remove('active');
    });
});

Probleme rencontre

Le menu burger marchait bien pour ouvrir/fermer, mais quand on cliquait sur un lien du menu en version mobile, le menu restait ouvert. Ca faisait bizarre. J'ai resolu ca en ajoutant un addEventListener sur chaque lien pour retirer la classe active au clic.

Extrait de code - Grille responsive avec CSS Grid

CSSstyle.css
/* Grille pour afficher les services de l'agence */
/* J'utilise auto-fit pour que ca s'adapte tout seul */
/* sans avoir besoin de media queries partout */
.services-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
    gap: 24px;
}

/* Pour les petits ecrans, une seule colonne */
@media (max-width: 768px) {
    .services-grid {
        grid-template-columns: 1fr;
    }
}

Probleme rencontre

Le site etait lent a charger, le score Lighthouse etait a 62/100. Apres avoir regarde ce qui posait probleme, c'etait les images. Elles etaient en PNG haute resolution, sans aucune compression.

Ce que j'ai fait : j'ai converti les images en WebP (format plus leger), j'ai ajoute loading="lazy" pour charger les images seulement quand on scroll dessus, et j'ai mis les attributs width et height pour eviter les sauts de page. Le score est passe a 94/100.

Extrait de code - Requete preparee en PHP

PHPservices.php
// Recuperation des services par categorie
// J'utilise une requete preparee pour eviter les injections SQL
// c'est un truc qu'on a bien vu en cours sur la securite
$stmt = $pdo->prepare("
    SELECT id, nom, description, prix
    FROM services
    WHERE categorie = :cat
    ORDER BY nom ASC
");

// On passe la categorie en parametre, pas directement dans la requete
$stmt->execute(['cat' => $categorie]);
$services = $stmt->fetchAll(PDO::FETCH_ASSOC);

Probleme rencontre

Au debut, le front en Next.js n'arrivait pas a communiquer avec l'API PHP. Les requetes fetch renvoyaient des erreurs. En cherchant sur Stack Overflow, j'ai compris que c'etait un probleme de CORS : le serveur PHP ne renvoyait pas les bons headers pour autoriser les requetes depuis le front.

Ce que j'ai fait : j'ai ajoute les headers Access-Control-Allow-Origin et Access-Control-Allow-Methods dans le fichier PHP de l'API. C'est pas complique quand on sait ce qu'il faut faire, mais il m'a fallu un moment pour comprendre le probleme.

Exemple concret : IntersectionObserver

En cherchant comment animer des elements au scroll, je suis tombe sur l'API IntersectionObserver via un article sur MDN et une video de Kevin Powell.

Avant, j'aurais utilise un ecouteur scroll avec des calculs de position, mais c'est pas performant parce que ca se declenche a chaque pixel de scroll. IntersectionObserver est beaucoup mieux parce que c'est le navigateur qui gere la detection.

Je l'ai utilise dans ce portfolio pour faire apparaitre les sections au fur et a mesure qu'on descend dans la page.
