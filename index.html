🔹 Projet EGNARO — Hébergement site web

Sept. 2024 → Jan. 2025

📌 Contexte

Dans le cadre du BTS SIO, nous devions héberger et déployer le site web du projet EGNARO sur un serveur. L'objectif était de mettre en place une infrastructure fonctionnelle et sécurisée.

👤 Ma contribution

  • Configuration du serveur Apache sous Debian
  • Mise en place de la base de données MySQL
  • Déploiement du site via FTP puis migration vers Git
  • Rédaction de la documentation d'installation

🛠️ Technologies utilisées

Apache 2.4 MySQL 8 PHP 8.1 FileZilla Git Debian

📷 Ajoute ici une capture d'écran du site EGNARO déployé

💻 Extrait de code — Connexion BDD sécurisée

PHPconnexion.php
// Connexion PDO sécurisée à la base de données
try {
    $pdo = new PDO(
        'mysql:host=localhost;dbname=egnaro;charset=utf8mb4',
        $user,
        $password,
        [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION]
    );
} catch (PDOException $e) {
    error_log('Erreur BDD : ' . $e->getMessage());
    die('Erreur de connexion.');
}

→ J'ai choisi PDO plutôt que MySQLi pour sa portabilité et la gestion native des requêtes préparées.

⚠️ Difficulté rencontrée & solution

Problème : Erreur 403 Forbidden après le déploiement sur le serveur Apache.

Diagnostic : Le fichier .htaccess n'avait pas les bonnes permissions et la directive AllowOverride était désactivée dans la config Apache.

Solution : chmod 644 .htaccess + activation de AllowOverride All dans le VirtualHost.

🚀 Mise à disposition du service

Procédure de déploiement suivie pour mettre le site en production :

  1. 1. Préparation serveur — Installation d'Apache, PHP et MySQL sur Debian. Configuration du VirtualHost avec le nom de domaine.
  2. 2. Transfert des fichiers — Déploiement initial via FTP (FileZilla), puis migration vers Git pour le versioning.
  3. 3. Configuration BDD — Import du schéma SQL, création de l'utilisateur dédié avec droits limités.
  4. 4. Tests de validation — Vérification des pages, formulaires, et connexion BDD. Test de charge basique.
  5. 5. Documentation — Rédaction d'un guide d'installation pour permettre la maintenance par un tiers.

💻 Extrait de code — Section responsive avec CSS Grid

CSSstyle.css
/* Grille responsive pour les services */
.services-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
    gap: 24px;
}

/* Adaptation mobile-first */
@media (max-width: 768px) {
    .services-grid {
        grid-template-columns: 1fr;
    }
}

→ Utilisation de CSS Grid avec auto-fit pour un layout naturellement responsive sans media queries excessive.

⚠️ Difficulté rencontrée & solution

Problème : Score Lighthouse Performance à 62/100 à cause d'images non optimisées.

Diagnostic : Les images étaient en PNG haute résolution sans compression ni attribut loading="lazy".

Solution : Conversion en WebP, ajout de loading="lazy" et width/height explicites. Score remonté à 94/100.

💻 Extrait de code — Requête SQL sécurisée

PHPservices.php
// Récupération des services avec requête préparée
$stmt = $pdo->prepare("
    SELECT id, nom, description, prix
    FROM services
    WHERE categorie = :cat
    ORDER BY nom ASC
");
$stmt->execute(['cat' => $categorie]);
$services = $stmt->fetchAll(PDO::FETCH_ASSOC);

→ Utilisation systématique de requêtes préparées pour prévenir les injections SQL.

⚠️ Difficulté rencontrée & solution

Problème : Conflit de versions entre le front-end Next.js et l'API PHP existante.

Diagnostic : Les headers CORS n'étaient pas configurés côté PHP, bloquant les requêtes fetch du front.

Solution : Ajout des headers Access-Control-Allow-Origin et Access-Control-Allow-Methods dans le fichier PHP d'API.

💡 Exemple concret de veille appliquée

Sujet : L'API IntersectionObserver pour les animations au scroll.

Source : Article MDN + tutoriel Kevin Powell (YouTube).

Application : J'ai utilisé cette API dans ce portfolio pour animer l'apparition des sections au défilement (classes .reveal-hidden / .reveal-visible), remplaçant une solution basée sur des écouteurs scroll moins performante.

Bénéfice : Meilleure performance (l'observateur natif du navigateur est plus efficace que des calculs manuels à chaque événement scroll), code plus propre et maintenable.
